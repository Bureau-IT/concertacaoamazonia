  1. Formulário de contato Gravity Forms (linhas 197-211)

  O que faz: Adiciona CSS inline apenas nas páginas "contato" ou "contact" para estilizar os formulários Gravity Forms (IDs 1 e 2).

  Funcionalidades:
  - Esconde labels dos campos (usa placeholders ao invés de labels visíveis)
  - Adiciona ícones SVG como background nos campos de input:
    - input_X_1: Ícone de pessoa (campo nome)
    - input_X_2: Ícone de prédio (campo empresa/organização)
    - input_X_3: Ícone de envelope (campo email)
  - Estiliza o botão de envio:
    - Largura 40%, cores usando variáveis globais do Elementor
    - Efeito hover invertendo as cores (fundo ↔ texto)

  Cores usadas:
  - --e-global-color-5376d26: Fundo do botão
  - --e-global-color-e978a34: Borda e texto do botão

  2. Bug fix Elementor Pro SVG (linhas 228-241)

  O que faz: Corrige um bug do Elementor Pro versão 3.33.x onde o módulo Off-Canvas quebra ao processar links dentro de elementos SVG.

  O problema:
  - Quando você tem um link (<a>) dentro de um SVG, a propriedade .href retorna um objeto SVGAnimatedString ao invés de uma string simples
  - O código do Elementor tenta chamar .includes() nesse objeto
  - SVGAnimatedString não tem o método .includes(), causando TypeError

  A solução:
  SVGAnimatedString.prototype.includes = function(searchString) {
      return this.baseVal.includes(searchString);
  };
  Adiciona o método .includes() ao prototype de SVGAnimatedString, delegando para baseVal (que é a string real).

  Quando afeta: Sites que usam Off-Canvas do Elementor Pro com SVGs clicáveis (ícones de redes sociais em SVG, por exemplo).
