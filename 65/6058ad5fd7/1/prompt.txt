# Plano: Corrigir renderização do Logo Concertação no bit-elementor-svg-widget

## Contexto

O widget `bureau_svg` (mu-plugin `bit-elementor-svg-widget.php`) está inserido no header do site concertacao com o SVG `logo-concertacao` selecionado. O SVG IS renderizado no DOM (46KB, presente), mas **completamente invisível** por dois motivos:

### Problema 1 — CSS global conflict (crítico)
Quando SVGs são inlined em HTML, seus `<style>` blocks se tornam CSS global. O SVG do logo define `.cls-1 { fill: currentColor; }`, mas **outro SVG na mesma página** (das 17 SVGs no header) define `.cls-1 { fill: none; }`. O segundo vence na cascata → paths do logo ficam `fill: none` (transparentes).

Evidência:
```
CSS rule 1: .cls-1 { fill: currentcolor }  ← logo SVG
CSS rule 2: .cls-1 { fill: none }          ← outro SVG (vence)
computedFill dos paths: "none"              ← resultado
```

### Problema 2 — SVG sem dimensões (secundário)
O SVG tem `viewBox="0 0 841.89 588.28"` mas sem `width`/`height` → expande para preencher o container: **1139×796px**. Precisa de controle de tamanho no widget.

## Solução

### Fix 1 — Auto-scoping de CSS no `render()`

Em `render()`, após `file_get_contents()`, injetar:
1. Atributo `data-scope="bsvg-{widget_id}"` no elemento `<svg>`
2. Prefixar as regras `.cls-*` no `<style>` com `[data-scope="bsvg-{widget_id}"]`
3. **Preservar** as regras `html[lang]` sem modificação (precisam referenciar o `<html>` global)

```php
// Após carregar o SVG e antes da injeção de class/id:
$scope_attr = 'bsvg-' . $this->get_id();

// Injetar data-scope no <svg>
$svg = preg_replace( '/<svg\b/', '<svg data-scope="' . $scope_attr . '"', $svg, 1 );

// Scopar CSS interno (apenas regras .cls-*, não as regras html[lang])
$svg = preg_replace_callback(
    '/(<style[^>]*>)(.*?)(<\/style>)/is',
    function ( $m ) use ( $scope_attr ) {
        $scoped = preg_replace(
            '/(?<!\w)(\.(?:cls|en|pt)-[\w-]+)\s*\{/',
            '[data-scope="' . $scope_attr . '"] $1 {',
            $m[2]
        );
        return $m[1] . $scoped . $m[3];
    },
    $svg
);
```

> Nota: a regex `(?<!\w)(\.cls-...)` evita modificar seletores que já têm prefixo.

### Fix 2 — Controle de largura no widget

Em `register_controls()`, adicionar no `content_section` após `svg_name`:

```php
$this->add_control( 'svg_width', [
    'label'      => 'Largura',
    'type'       => \Elementor\Controls_Manager::SLIDER,
    'size_units' => [ 'px', '%', 'vw' ],
    'range'      => [
        'px' => [ 'min' => 50, 'max' => 800, 'step' => 10 ],
        '%'  => [ 'min' => 5,  'max' => 100, 'step' => 5  ],
    ],
    'default'    => [ 'unit' => 'px', 'size' => 200 ],
    'selectors'  => [
        '{{WRAPPER}} svg' => 'width: {{SIZE}}{{UNIT}}; height: auto;',
    ],
] );
```
