# Plano: Migrar Smart Filters para Atlas Cultural /cultura/

## Contexto

A página "Atlas Cultural das Amazônias" (post 57548, subsite blog_id=2 `/cultura/`) tem widgets de filtro JetSmartFilters configurados com IDs do **site raiz** (blog_id=1). No admin do subsite `/cultura/`, esses filtros não aparecem na lista porque o JetSmartFilters usa WPML para listar os filtros via JOIN com `wp_2_icl_translations`.

### Causa raiz confirmada

O JetSmartFilters (v3.7.5) usa esta query para listar os filtros no admin:
```sql
SELECT posts.ID, ... FROM wp_2_posts
LEFT JOIN wp_2_icl_translations ON posts.ID = translations.element_id
WHERE posts.post_type = 'jet-smart-filters'
AND translations.language_code = 'pt-br'
```

Os filtros 89876-89881 existem em `wp_2_posts` e `wp_2_postmeta` com configuração correta, mas **não têm entrada em `wp_2_icl_translations`**. Por isso o LEFT JOIN os filtra pela condição `language_code = 'pt-br'` — sem registro WPML, ficam invisíveis.

O único filtro visível (90039) tem seu registro WPML:
`(translation_id=2003, element_type='post_jet-smart-filters', element_id=90039, trid=2138390, language_code='pt-br', source_language_code=NULL)`

Os filtros equivalentes **já foram criados** no subsite cultura com novos IDs (89876-89881) e estão corretamente configurados em `wp_2_postmeta`. O trabalho necessário é:
