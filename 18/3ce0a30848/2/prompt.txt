# Plano: Configurar segLinks no widget Espiral do Conhecimento

## Context

A espiral do conhecimento é renderizada pelo widget Elementor `bit-elementor-espiral-widget.php` em duas páginas:
- Home do site
- `/conhecimento/espiral-de-conhecimento/`

O widget carrega o SVG `espiral-do-conhecimento.svg` do tema child e injeta os links via `apply_config_to_svg()`, usando um JSON salvo na option `bit_espiral_config` no banco.

**Problema:** A option `bit_espiral_config` não existe ainda (`Error: Could not get 'bit_espiral_config' option. Does it exist?`). Os links estão hardcoded no SVG do tema (`espiral-do-conhecimento.svg`), mas o SVG bundled no plugin (`espiral-2025.svg`) tem apenas `href="#"`. O widget sobrescreve os hrefs do SVG via `segLinks` no JSON — portanto, **os links hardcoded no SVG do tema não têm efeito quando o widget está ativo**.

**Conclusão:** É preciso criar a option `bit_espiral_config` com os `segLinks` corretos para os 21 segmentos. Os links do SVG do tema foram corrigidos no commit anterior, mas são ignorados pelo widget. O canal correto é o JSON da option.
