# Skill: save-report

## Quando usar

Sempre que um relatório for gerado durante uma conversa — seja de saúde de infraestrutura, análise de performance, diagnóstico, otimização, migração, ou qualquer documento de entrega técnica.

## Estrutura de diretórios obrigatória

```
~/scripts/reports/[área bureau]/[nome do programa]/relatorio-[tipo]-YYYY-MM-DD.[ext]
```

## Áreas bureau válidas

| Código | Nome |
|--------|------|
| `01 - Governança` | Políticas, processos, compliance |
| `02 - Infraestrutura` | AWS, servidores, redes, banco de dados |
| `03 - Web` | Sites, deploys, frontend, CDN |
| `04 - Consultoria e Suporte` | Tickets, suporte a clientes |
| `05 - Ambiente: Educação, Cultura e Treinamento` | Documentação educacional, tutoriais |
| `06 - Comercialização Software e Hardware` | Licenças, produtos, cotações |
| `07 - Automação` | Scripts, pipelines, CI/CD, bots |
| `08 - Segurança` | Auditorias, firewall, incidentes, pentest |

## Regras de nomenclatura

- Formato obrigatório: `relatorio-[tipo]-YYYY-MM-DD.[ext]`
- Tipos comuns: `saude`, `migracao`, `otimizacao`, `diagnostico`, `auditoria`, `performance`
- Extensões: `.html` (principal), `.pdf` (quando possível), `.md` (alternativo)
- Usar data real do dia em que o relatório foi gerado (não a data do período analisado)

## Exemplos de caminhos válidos

```
~/scripts/reports/02 - Infraestrutura/plesk/relatorio-saude-ec2-2026-03-15.html
~/scripts/reports/02 - Infraestrutura/plesk/relatorio-saude-ec2-2026-03-15.pdf
~/scripts/reports/08 - Segurança/cloudflare/relatorio-auditoria-waf-2026-03-20.html
~/scripts/reports/03 - Web/mfb/relatorio-performance-2026-04-01.html
~/scripts/reports/07 - Automação/backup-s3/relatorio-diagnostico-2026-03-10.md
```

## Processo de execução

1. **Identificar área bureau** com base no contexto do relatório
2. **Identificar nome do programa** — usar o diretório do projeto em `~/scripts/` (ex: `plesk`, `mfb`, `elos-dev`)
3. **Criar diretório** se não existir:
   ```bash
   mkdir -p "~/scripts/reports/[área]/[programa]"
   ```
4. **Salvar HTML** com design consistente Bureau (fundo escuro, glassmorphism):
   - Fundo: `#0a0a0b`
   - Glassmorphism: `backdrop-filter: blur(12px)`
   - Fontes: Inter + JetBrains Mono
   - Floating orbs, animações CSS
   - Footer com "Bureau de Tecnologia"
5. **Converter para PDF** via Playwright MCP quando disponível:
   ```
   mcp__playwright__playwright_navigate → file:///Users/dcambria/scripts/reports/[área]/[programa]/[arquivo].html
   mcp__playwright__playwright_save_as_pdf → [mesmo caminho].pdf
   ```
6. **Confirmar** ao usuário com os caminhos dos arquivos gerados

## Design padrão Bureau

Os relatórios devem seguir o design das páginas do Plesk (default, maintenance, 404, suspended):
- Background escuro (`#0a0a0b`) com gradientes animados
- Glassmorphism (`backdrop-filter: blur`)
- Status dots coloridos (verde OK, amarelo atenção, vermelho erro)
- Tabelas com hover sutil
- Badges de área e data no header
- Responsivo para mobile
